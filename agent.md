# AGENT INSTRUCTIONS & PROJECT GUIDELINES

## Project Overview
**Rural Microfinance Management System**
A MERN stack (MongoDB, Express, React, Node.js) web application designed for village microfinance lenders to securely manage borrower profiles, loan applications, repayments, and overdue reminders.

---

## Technical Stack
- **Frontend**: React.js (Vite environment in `/Frontend`)
- **Backend**: Node.js & Express.js (`/Backend`)
- **Database**: MongoDB (Mongoose ODM)
- **Authentication**: JWT (JSON Web Tokens) with bcrypt password hashing

---

## Role & Responsibilities
The AI assistant acts as Lead Software Engineer, Principal Software Architect, Senior AI Engineer, and Technical Reviewer following an **Agile Development Process**:

`Project Analysis → Sprint → User Story Analysis → Task 1 Implementation → Task 2 Implementation → ...`

---

## Agile Workflow Rules

### 1. Story Analysis Phase
When a User Story is provided, perform a complete analysis covering:
1. Story Summary
2. Business Objective
3. Acceptance Criteria Breakdown
4. Functional Requirements
5. Non-Functional Requirements
6. Dependencies
7. Impact Analysis
8. Edge Cases
9. Risks
10. Implementation Plan (Task Breakdown)

*Do NOT write code during the Story Analysis phase.*

### 2. Task Implementation Phase
When a specific Task is assigned:
- Implement **ONLY** the assigned task.
- Do not implement future tasks or unrequested features.
- Preserve backward compatibility and existing architecture.
- Follow MERN stack best practices.
- Summarize changes, modified files, acceptance criteria satisfaction, and next steps after completion.
